==================================================
OVERVIEW
==================================================
This backend wraps part of the Upload-Post API.

Two token types exist:
- Upload-Post API key:
  Used by the backend for account, profile, history, and comments endpoints
- Upload-Post JWT from access_url?token=...:
  Generated by jwt/generate
  Intended for JWT validation and some analytics flows

Important:
- Do not confuse API key and JWT
- In practice, based on current testing, profile analytics is working with the Upload-Post API key flow on the backend

==================================================
WORKING ENDPOINTS
==================================================
1. GET /api/v1/upload-post/account/me
Purpose:
- Check current Upload-Post account

Thunder Client:
- Auth = None
- Headers = none
- Body = empty

2. POST /api/v1/upload-post/users
Purpose:
- Create a profile

Thunder Client:
- Auth = None
- Headers:
  Content-Type = application/json
- Body JSON:
{
  "username": "blhoang23"
}

3. GET /api/v1/upload-post/users
Purpose:
- List all profiles created in Upload-Post

Thunder Client:
- Auth = None
- Headers = none
- Body = empty

4. GET /api/v1/upload-post/users/{username}
Purpose:
- Get one profile

Example:
- GET /api/v1/upload-post/users/blhoang23

Thunder Client:
- Auth = None
- Headers = none
- Body = empty

5. POST /api/v1/upload-post/jwt/generate
Purpose:
- Generate Upload-Post connect link
- Limit shown platforms with platforms array

Thunder Client:
- Auth = None
- Headers:
  Content-Type = application/json
- Body JSON:
{
  "username": "blhoang23",
  "platforms": ["youtube", "tiktok"]
}

Result:
- Returns access_url
- User must open access_url in browser and complete social connection there

6. GET /api/v1/upload-post/analytics/profiles/{username}
Purpose:
- Get profile analytics

Example:
- GET /api/v1/upload-post/analytics/profiles/blhoang23?platforms=youtube,tiktok

Thunder Client:
- Auth = Bearer
- Bearer Token = use the token required by your working environment
- Headers = none
- Body = empty

Test note:
- In actual testing, this flow appears to be authenticated effectively through the Upload-Post API key behavior on the backend side
- Keep this endpoint marked as usable, but verify auth behavior carefully during demo

7. GET /api/v1/upload-post/history
Purpose:
- Get Upload-Post upload history

Example:
- GET /api/v1/upload-post/history?page=1&limit=20

Thunder Client:
- Auth = None
- Headers = none
- Body = empty

8. GET /api/v1/upload-post/interactions/comments
Purpose:
- Get comments for supported platforms

Example by post_url:
- GET /api/v1/upload-post/interactions/comments?platform=instagram&user=blhoang23&post_url=https://www.instagram.com/p/ABC123/

Example by post_id:
- GET /api/v1/upload-post/interactions/comments?platform=instagram&user=blhoang23&post_id=123456789

Thunder Client:
- Auth = None
- Headers = none
- Body = empty

Required query params:
- platform
- user
- one of:
  post_id
  post_url

==================================================
LIMITATIONS FOUND DURING TESTING
==================================================
1. Validate JWT endpoint is not reliable
- Endpoint:
  POST /api/v1/upload-post/jwt/validate
- Current note:
  Do not rely on this endpoint for the demo
- Reason:
  It appears to return errors consistently in current testing

2. Profile analytics auth behavior is confusing
- Endpoint:
  GET /api/v1/upload-post/analytics/profiles/{username}
- Current note:
  Treat this as working through the backend Upload-Post integration, but do not over-explain JWT validation in the demo

3. Total impressions is currently not working
- Endpoint:
  GET /api/v1/upload-post/analytics/profiles/{username}/total-impressions
- Current note:
  Currently returns 500 Internal Server Error
- This still failed even when testing with:
  API key
  JWT generated from the connect flow
- Recommendation:
  Mark as known limitation / upstream issue for now

4. History only shows Upload-Post-managed uploads
- Endpoint:
  GET /api/v1/upload-post/history
- Current note:
  This does not list every original post from the connected social account
- It only lists posts uploaded or tracked through Upload-Post

5. Comments endpoint does not support YouTube
- Endpoint:
  GET /api/v1/upload-post/interactions/comments
- Current note:
  Do not use platform=youtube
- Working expectation:
  Use only on platforms supported by Upload-Post comments API, such as Instagram if available in your account setup

- GET /api/v1/upload-post/history
- GET /api/v1/upload-post/interactions/comments

- POST /api/v1/upload-post/jwt/validate
- GET /api/v1/upload-post/analytics/profiles/{username}/total-impressions

==================================================
QUICK AUTH RULES
==================================================
Use Auth = None for:
- /account/me
- /users
- /users/{username}
- /history
- /interactions/comments
- /jwt/generate
- /jwt/validate

Use Auth = Bearer only when needed for analytics route testing:
- /analytics/profiles/{username}
- /analytics/profiles/{username}/total-impressions

Important:
- Even if a JWT is available, some provider behavior is inconsistent
- Follow the tested working flow instead of assuming the docs are fully accurate
