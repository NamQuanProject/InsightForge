INSIGHTFORGE API DOCS
FastAPI + A2A Agents + Supabase Postgres + Upload-Post

Base FastAPI URL:
http://127.0.0.1:8000

==================================================
1. SYSTEM OVERVIEW
==================================================

Current system has 4 runtime parts:

1. Trend Agent
- A2A server
- default URL: http://localhost:9997
- analyzes trends and returns trend_analysis JSON

2. Content Agent
- A2A server
- default URL: http://localhost:9998
- receives trend context and returns generated_content JSON

3. Orchestrator
- A2A server
- default URL: http://localhost:9996
- routes user prompt to Trend Agent and Content Agent
- final output shape:
{
  "trend_analysis": {},
  "generated_content": {}
}

4. FastAPI Backend
- API server
- default URL: http://127.0.0.1:8000
- exposes normal HTTP endpoints for frontend / Thunder Client
- new orchestrator endpoint calls the running A2A orchestrator

Important:
- The orchestrator endpoint does not replace the agents.
- You must run the 3 agent servers first, then run FastAPI.
- Supabase is used as PostgreSQL through SQLAlchemy.

==================================================
5. AGENT ORCHESTRATOR API
==================================================

Route prefix:
/api/v1/agents

----------------------------------------
1. AGENTS STATUS
----------------------------------------

GET /api/v1/agents/status

Purpose:
- check whether the 3 A2A servers are reachable from FastAPI

Response:
{
  "status": "ok",
  "processes": [
    {
      "name": "routing_orchestrator",
      "url": "http://localhost:9996",
      "reachable": true,
      "detail": "reachable"
    },
    {
      "name": "trend_agent",
      "url": "http://localhost:9997",
      "reachable": true,
      "detail": "reachable"
    },
    {
      "name": "content_agent",
      "url": "http://localhost:9998",
      "reachable": true,
      "detail": "reachable"
    }
  ]
}

Field meanings:
- status: ok if all processes are reachable, degraded otherwise
- name: process label used by backend
- url: A2A server URL
- reachable: true if socket connection succeeds
- detail: readable status text

----------------------------------------
2. ORCHESTRATE TREND + CONTENT
----------------------------------------

POST /api/v1/agents/orchestrate

Request:
{
  "prompt": "Chu de dang trend trong ngay hom nay ve the thao la gi va toi nen tao noi dung nhu the nao?",
  "save_files": true,
  "user_id": null
}

Request fields:
- prompt: user prompt sent to the routing orchestrator
- save_files: if true, backend saves raw and extracted outputs to local JSON files
- user_id: optional app user UUID. If provided, saved trend/content rows are linked to this user

Response:
{
  "status": "success",
  "trend_analysis_id": "uuid",
  "generated_content_id": "uuid",
  "output": {
    "trend_analysis": {
      "query": "string",
      "results": [],
      "markdown_summary": "string"
    },
    "generated_content": {
      "selected_keyword": "string",
      "main_title": "string",
      "video_script": {},
      "platform_posts": {},
      "thumbnail": {},
      "music_background": "string"
    }
  },
  "raw_response": {},
  "raw_response_file": "orchestrator_raw_response.json",
  "output_file": "orchestrator_output.json"
}

Response fields:
- status: backend request status
- trend_analysis_id: Supabase/Postgres id for the saved trend_analyses row
- generated_content_id: Supabase/Postgres id for the saved generated_contents row
- output: extracted final business JSON from orchestrator
- output.trend_analysis: trend agent result block
- output.generated_content: content agent result block
- raw_response: full A2A JSON-RPC response, useful for debugging
- raw_response_file: local file path for full A2A response when save_files=true
- output_file: local file path for extracted final JSON when save_files=true

Saved files:
- orchestrator_raw_response.json
- orchestrator_output.json

Notes:
- This endpoint can take several minutes.
- Trend Agent and Content Agent are called sequentially.
- Recommended client timeout: 420 seconds.
- Backend normalizes output before saving, so output_file always has:
  - trend_analysis
  - generated_content
- Even if the model returns raw text, backend wraps it into the expected shape and marks errors where needed.
- This endpoint saves to Supabase tables:
  - trend_analyses
  - generated_contents

==================================================
12. COMMON ISSUES
==================================================

1. Port already in use

Check:
netstat -ano | findstr :9996
netstat -ano | findstr :9997
netstat -ano | findstr :9998
netstat -ano | findstr :8000

Stop process:
Stop-Process -Id <PID>

2. Agents status degraded

Make sure all 3 agent terminals are running:
- trend agent
- content agent
- orchestrator

3. Orchestrator timeout

This pipeline can take several minutes.
Use timeout >= 420 seconds.

4. Output file still old

Delete or ignore old:
- orchestrator_raw_response.json
- orchestrator_output.json

Then run the request again.

5. Model/key issue

Current default is Gemini:
- ROUTING_AGENT_MODEL=gemini:gemini-2.5-flash
- TREND_AGENT_MODEL=gemini/gemini-2.5-flash
- CONTENT_AGENT_MODEL=gemini/gemini-2.5-flash

Make sure GOOGLE_API_KEY or GEMINI_API_KEY exists in .env.
